# Patent Writing Assistant

## 核心能力

1. **检索能力** - 多源检索专利、论文、百科、法律条文
2. **学习模仿能力** - 分析优秀文章风格并生成风格指南
3. **撰写能力** - 撰写专利申请文件、技术文档、研究报告
4. **检查校对能力** - 灵活的校对功能，支持用户自定义校对方向

## 触发词

| 模块 | 触发词 |
|------|--------|
| search | "检索"、"搜索"、"查找"、"查询"、"找一下" |
| learn | "学习"、"模仿"、"分析风格"、"参考写法" |
| write | "撰写"、"写"、"起草"、"生成"、"编写" |
| proofread | "检查"、"校对"、"审核"、"验证"、"修改建议" |

## 工作流程

### 流程 1：撰写专利文件（核心流程）

```
用户上传技术交底文件
    ↓
[步骤1] 资料检索（强制前置）
  - 检索优先级：相关专利 > 相关论文 > 百科知识
  - 保存检索结果到 workspace/search-results/
  - 用户确认：✅ 确认检索结果
    ↓
[步骤2] 生成技术方案
  - 基于交底文件 + 检索结果
  - 用户确认：✅ 确认技术方案（关键节点）
    ↓
[步骤3] 生成权利要求书
  - 基于技术方案 + 检索结果
  - 用户确认：✅ 确认权利要求书（关键节点）
    ↓
[步骤4] 生成背景技术
  - 基于权利要求书 + 检索结果
  - 无需用户确认（自动进行）
    ↓
[步骤5] 生成发明内容/实用新型内容
  - 基于技术方案 + 权利要求书
  - 无需用户确认（自动进行）
    ↓
[步骤6] 判断是否需要附图
  - 如果需要 → AI生成附图描述（用户自己画图）
  - 用户确认：✅ 确认附图（关键节点）
    ↓
[步骤7] 生成具体实施方式
  - 基于技术方案 + 附图描述 + 权利要求书
  - 无需用户确认（自动进行）
    ↓
[步骤8] 整合输出
  - 整合所有部分为完整专利申请文件
  - 转换为Word格式
  - 用户确认：✅ 最终确认（关键节点）
```

### 流程 2：学习风格

```
用户输入：文章样本/领域关键词
    ↓
[步骤1] 文章收集
  - 用户指定文件 → 直接使用
  - 用户指定领域 → AI自动搜索收集
    ↓
[步骤2] 表面特征分析
  - 句式结构、段落组织、用词习惯、格式规范
    ↓
[步骤3] 深层特征分析
  - 逻辑链条、论证方式、语言风格、专业表达
    ↓
[步骤4] 生成风格指南
  - 保存到 workspace/style-guides/
    ↓
[步骤5] 应用风格指南
  - 撰写时询问用户是否应用
```

### 流程 3：检查校对

```
用户输入：待校对文档
    ↓
[步骤1] 询问校对方向
  - 用词规范、文章结构、用句要求、语法检查等
    ↓
[步骤2] 执行校对
  - 按用户选择的维度进行校对
    ↓
[步骤3] 输出结果
  - 校对报告 + 修改建议
```

### 流程 4：单独检索

```
用户输入：检索关键词/主题
    ↓
[步骤1] 执行检索
  - 专利检索、论文检索、百科检索
    ↓
[步骤2] 整理结果
  - 按来源分类整理
  - 保存到 workspace/search-results/
    ↓
[步骤3] 输出检索报告
```

## 模块调用规则

| 规则 | 说明 |
|------|------|
| **检索前置** | 撰写前必须先检索，确保有足够信息 |
| **学习可选** | 用户提供样本时才触发学习模块 |
| **校对必做** | 每次撰写后必须进行校对 |
| **独立调用** | 每个模块可以单独使用 |
| **组合调用** | 多个模块可以按需组合使用 |

## 用户确认节点

| 节点 | 用户操作 | AI操作 |
|------|----------|--------|
| **步骤1后** | 确认检索结果 | 展示检索结果，询问是否需要补充 |
| **步骤2后** | 确认技术方案 | 展示技术方案，等待确认 |
| **步骤3后** | 确认权利要求书 | 展示权利要求书，等待确认 |
| **步骤6后** | 确认附图 | 展示附图描述，等待确认 |
| **步骤8后** | 确认终稿 | 展示完整专利文件，等待最终确认 |
