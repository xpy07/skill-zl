# 背景技术模板

## 基本格式

```markdown
# 背景技术

本发明涉及[技术领域]，具体涉及[具体技术方向]。

现有技术中，已有相关专利/论文，例如：

[专利号/论文标题1]：[简要描述该现有技术的技术方案]

[专利号/论文标题2]：[简要描述该现有技术的技术方案]

[专利号/论文标题3]：[简要描述该现有技术的技术方案]

但是，上述现有技术存在以下缺点：

1. [缺点1]：[详细说明]

2. [缺点2]：[详细说明]

3. [缺点3]：[详细说明]

因此，需要一种[技术方案]，以解决上述技术问题。
```

## 撰写规则

### 引用现有技术

**要求**：
- 引用相关的专利/论文
- 简要描述现有技术的技术方案
- 说明现有技术与本发明的关系

**引用格式**：
- 专利：[专利号]公开了一种...，该...包括...
- 论文：[作者]在[论文标题]中提出...，该方案...
- 产品：现有产品采用...，其特点在于...

### 分析缺点

**要求**：
- 客观分析现有技术的缺点
- 不贬低现有技术
- 说明缺点的技术影响

**常见缺点类型**：
- 效率低：处理速度慢、响应时间长
- 成本高：制造成本高、维护成本高
- 体积大：占用空间大、不便于携带
- 功能单一：只能实现单一功能
- 兼容性差：与其他设备不兼容
- 可靠性低：容易出现故障
- 用户体验差：操作复杂、界面不友好

### 引出本发明

**要求**：
- 自然引出本发明
- 说明本发明要解决的问题
- 为发明内容做铺垫

## 引用来源

### 专利引用

**格式**：
```
中国专利公开号CN[专利号]A，公开了一种[技术方案]，该[技术方案]包括...
```

**来源**：
- CNIPA（中国国家知识产权局）
- Google Patents
- USPTO（美国专利商标局）

### 论文引用

**格式**：
```
[作者]在《[论文标题]》[期刊/会议名称]（[年份]）中提出...
```

**来源**：
- 知网（CNKI）
- 万方
- Semantic Scholar
- arXiv

### 产品引用

**格式**：
```
现有产品（如[产品名称]）采用...，其特点在于...
```

**来源**：
- 产品说明书
- 技术文档
- 用户手册

## 常见错误

| 错误类型 | 示例 | 正确做法 |
|----------|------|----------|
| 引用不相关 | 引用与本发明无关的专利/论文 | 只引用相关的专利/论文 |
| 贬低现有技术 | 现有技术非常落后 | 客观描述现有技术的缺点 |
| 缺少引用 | 未引用现有技术 | 引用相关的专利/论文 |
| 缺点描述不清 | 只说"效果不好" | 详细说明缺点的技术影响 |
