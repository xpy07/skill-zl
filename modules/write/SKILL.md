# 撰写模块

## 功能说明

撰写专利申请文件，遵循7步流程，确保内容完整、格式规范。

## 触发词

"撰写"、"写"、"起草"、"生成"、"编写"

## 支持的文档类型

| 文档类型 | 说明 |
|----------|------|
| 发明专利申请 | 完整的发明专利申请文件 |
| 实用新型申请 | 完整的实用新型专利申请文件 |
| 技术文档 | 技术交底书、创新点分析等 |
| 研究报告 | 研究分析报告 |

## 撰写流程

### 步骤1：资料检索（强制前置）

| 项目 | 说明 |
|------|------|
| **输入** | 技术交底文件 |
| **检索优先级** | ① 相关专利 → ② 相关论文 → ③ 百科知识 |
| **输出** | 检索结果汇总文件 |
| **用户确认** | ✅ 确认检索结果 |
| **存储位置** | `workspace/search-results/{关键词}-{日期}.json` |

**执行逻辑**：
1. 解析技术交底文件，提取关键技术信息
2. 生成检索关键词（包括同义词扩展）
3. 按优先级检索：专利 → 论文 → 百科
4. 整理检索结果，保存到文件
5. 展示检索结果摘要，等待用户确认

---

### 步骤2：生成技术方案

| 项目 | 说明 |
|------|------|
| **输入** | 技术交底文件 + 检索结果 |
| **分析内容** | 技术领域、技术问题、技术方案、技术效果 |
| **输出** | 完整的技术方案文档 |
| **用户确认** | ✅ 确认技术方案（关键节点） |

**执行逻辑**：
1. 分析交底文件的技术要点
2. 结合检索结果补充技术细节
3. 整理成完整的技术方案
4. 包含：技术领域、技术问题、技术方案、技术效果
5. 展示技术方案，等待用户确认

**技术方案文档结构**：
```markdown
# 技术方案

## 1. 技术领域
[本发明所属的技术领域]

## 2. 技术问题
[本发明要解决的技术问题]

## 3. 技术方案
[本发明的技术方案详细描述]

### 3.1 核心技术点
[核心技术点1]
[核心技术点2]
[...]

### 3.2 技术细节
[技术细节详细描述]

## 4. 技术效果
[本发明的技术效果和优势]
```

---

### 步骤3：生成权利要求书

| 项目 | 说明 |
|------|------|
| **输入** | 技术方案 + 检索结果 |
| **分析内容** | 确定创新点、确定保护范围 |
| **撰写内容** | 独立权利要求 + 从属权利要求 |
| **输出** | 权利要求书 |
| **用户确认** | ✅ 确认权利要求书（关键节点） |

**执行逻辑**：
1. 分析技术方案，确定创新点
2. 确定保护范围（独立权利要求）
3. 撰写独立权利要求（前序部分 + 特征部分）
4. 撰写从属权利要求（梯度布局）
5. 检查格式规范（引用关系、术语一致性）
6. 展示权利要求书，等待用户确认

**权利要求书格式**：
```markdown
# 权利要求书

1. 一种[技术领域]，其特征在于，包括：
   [技术特征1]；
   [技术特征2]；
   [技术特征3]。

2. 根据权利要求1所述的[技术领域]，其特征在于，所述[技术特征1]包括：
   [进一步限定1]；
   [进一步限定2]。

3. 根据权利要求1或2所述的[技术领域]，其特征在于，所述[技术特征2]包括：
   [进一步限定3]。

[...继续从属权利要求...]
```

**权利要求撰写规则**：
- 独立权利要求：包含解决技术问题的全部必要技术特征
- 从属权利要求：对独立权利要求的进一步限定
- 引用关系：清晰、正确
- 术语一致性：全文术语统一

### 步骤4：生成背景技术

| 项目 | 说明 |
|------|------|
| **输入** | 权利要求书 + 检索结果 |
| **撰写内容** | 现有技术描述、现有技术缺点、本发明要解决的问题 |
| **引用要求** | 引用检索到的相关专利/论文 |
| **输出** | 背景技术部分 |
| **用户确认** | 无需确认（自动进行） |

**执行逻辑**：
1. 基于检索结果，描述现有技术
2. 分析现有技术的缺点和不足
3. 引用相关专利/论文作为现有技术
4. 说明本发明要解决的技术问题
5. 生成背景技术部分

**背景技术文档结构**：
```markdown
# 背景技术

[本发明涉及[技术领域]，具体涉及[具体技术方向]。]

[现有技术中，已有相关专利/论文，例如：]
- [专利号/论文标题1]：[简要描述]
- [专利号/论文标题2]：[简要描述]

[但是，现有技术存在以下缺点：]
1. [缺点1]
2. [缺点2]
3. [缺点3]

[因此，需要一种[技术方案]，以解决上述技术问题。]
```

---

### 步骤5：生成发明内容/实用新型内容

| 项目 | 说明 |
|------|------|
| **输入** | 技术方案 + 权利要求书 |
| **撰写内容** | 发明目的、技术方案、有益效果 |
| **格式要求** | 与权利要求书对应 |
| **输出** | 发明内容部分 |
| **用户确认** | 无需确认（自动进行） |

**执行逻辑**：
1. 基于技术方案和权利要求书
2. 撰写发明目的（解决的技术问题）
3. 撰写技术方案（与权利要求书对应）
4. 撰写有益效果（技术效果和优势）
5. 生成发明内容部分

**发明内容文档结构**：
```markdown
# 发明内容

## 1. 发明目的
[本发明要解决的技术问题是：...]

## 2. 技术方案
[为解决上述技术问题，本发明采用如下技术方案：]

[与权利要求书对应的技术方案描述]

## 3. 有益效果
[本发明具有以下有益效果：]
1. [效果1]
2. [效果2]
3. [效果3]
```

---

### 步骤6：判断是否需要附图

| 项目 | 说明 |
|------|------|
| **输入** | 技术方案 |
| **判断依据** | 是否有多个组件/步骤/连接关系 |
| **判断标准** | 如果有多个组件/步骤/连接关系 → 需要附图 |
| **附图类型** | 结构图、流程图、连接关系图 |
| **输出** | 附图描述（用户自己画图） |
| **用户确认** | ✅ 确认附图（关键节点） |

**执行逻辑**：
1. 分析技术方案的复杂度
2. 判断是否需要附图辅助说明
3. 如果需要：
   - 确定附图类型（结构图/流程图/连接关系图）
   - 生成附图描述（包含各部件标记）
   - 用户根据描述自己画图
4. 如果不需要：跳过此步骤

**附图描述格式**：
```markdown
# 附图说明

## 图1：[图的名称]
[图的详细描述，包括各部件标记]

标记说明：
- 10：[部件10的名称]
- 20：[部件20的名称]
- 30：[部件30的名称]
- [...]
```

---

### 步骤7：生成具体实施方式

| 项目 | 说明 |
|------|------|
| **输入** | 技术方案 + 附图描述 + 权利要求书 |
| **撰写内容** | 详细实施步骤、实施例、注意事项 |
| **格式要求** | 引用附图标记、支持所有权利要求 |
| **输出** | 具体实施方式部分 |
| **用户确认** | 无需确认（自动进行） |

**执行逻辑**：
1. 基于技术方案和附图描述
2. 详细描述如何实施该发明
3. 包含多个实施例（如果适用）
4. 引用附图标记
5. 确保支持所有权利要求
6. 生成具体实施方式部分

**具体实施方式文档结构**：
```markdown
# 具体实施方式

## 实施例1
[详细描述实施例1，引用附图标记]

如图1所示，[详细描述...]

其中，[部件10]用于[功能]，[部件20]用于[功能]，...

## 实施例2
[详细描述实施例2，引用附图标记]

如图2所示，[详细描述...]

## 实施例3（可选）
[详细描述实施例3]

## 注意事项
1. [注意事项1]
2. [注意事项2]
3. [注意事项3]
```

---

### 步骤8：整合输出

| 项目 | 说明 |
|------|------|
| **输入** | 所有部分内容 |
| **整合内容** | 技术领域 + 背景技术 + 发明内容 + 附图说明 + 具体实施方式 + 权利要求书 |
| **检查内容** | 格式规范、逻辑一致性、权利要求支持 |
| **输出格式** | Word文档 |
| **输出** | 完整专利申请文件（终稿） |
| **用户确认** | ✅ 最终确认（关键节点） |

**执行逻辑**：
1. 整合所有部分的内容
2. 检查格式规范
3. 检查逻辑一致性
4. 检查权利要求是否得到说明书支持
5. 转换为Word格式
6. 展示完整专利文件，等待用户最终确认

**完整专利申请文件结构**：
```markdown
# [发明名称]

## 技术领域
[步骤2生成的内容]

## 背景技术
[步骤4生成的内容]

## 发明内容
[步骤5生成的内容]

## 附图说明
[步骤6生成的内容]

## 具体实施方式
[步骤7生成的内容]

# 权利要求书
[步骤3生成的内容]

# 说明书摘要
[根据发明内容生成的摘要]
```

## 用户确认节点汇总

| 节点 | 用户操作 | AI操作 |
|------|----------|--------|
| **步骤1后** | 确认检索结果 | 展示检索结果，询问是否需要补充 |
| **步骤2后** | 确认技术方案 | 展示技术方案，等待确认 |
| **步骤3后** | 确认权利要求书 | 展示权利要求书，等待确认 |
| **步骤6后** | 确认附图 | 展示附图描述，等待确认 |
| **步骤8后** | 确认终稿 | 展示完整专利文件，等待最终确认 |

## 与学习模块的交互

在步骤2（生成技术方案）之前，检查是否有对应的风格指南：
- 如果有 → 展示风格指南摘要，询问用户是否应用
- 如果没有 → 询问用户是否需要学习风格

## 与校对模块的交互

在步骤8（整合输出）之后，自动调用校对模块：
- 询问用户校对方向
- 执行校对
- 生成校对报告

## 输出格式

最终输出为Word文档，保存位置：`workspace/documents/{文档名称}.docx`
