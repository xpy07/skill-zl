# 学习模块

## 功能说明

分析优秀文章风格，生成完整风格指南，供撰写时参考应用。

## 触发词

"学习"、"模仿"、"分析风格"、"参考写法"

## 学习领域

| 领域 | 学习重点 |
|------|----------|
| 专利申请文件 | 权利要求格式、说明书结构、背景技术写法、发明内容表达 |
| 技术文档 | 技术方案描述、术语解释方式、逻辑结构、图表配合 |
| 研究报告 | 研究背景、方法描述、结果呈现、讨论分析、结论表达 |
| 特定行业文档 | 行业术语、行业规范、行业特有的表达方式 |

## 学习流程

```
用户输入：文章样本/领域关键词
    ↓
[步骤1] 文章收集
  ├─ 方式1：用户指定文件（上传/粘贴文本）
  ├─ 方式2：用户指定领域，AI自动搜索收集
  └─ 方式3：混合模式（AI搜索 + 用户补充）
    ↓
[步骤2] 表面特征分析
  ├─ 句式结构分析
  │   - 长短句比例
  │   - 从句使用频率
  │   - 常见句式模板
  ├─ 段落组织分析
  │   - 段落长度分布
  │   - 主题句位置
  │   - 过渡方式
  ├─ 用词习惯分析
  │   - 高频词汇统计
  │   - 固定搭配识别
  │   - 专业术语提取
  └─ 格式规范分析
      - 标题层级结构
      - 编号方式
      - 引用格式
    ↓
[步骤3] 深层特征分析
  ├─ 逻辑链条分析
  │   - 论证结构（总分总、问题-解决、因果推理）
  │   - 因果关系表达方式
  │   - 逻辑连接词使用
  ├─ 论证方式分析
  │   - 数据支撑方式
  │   - 案例引用方式
  │   - 权威引用方式
  ├─ 语言风格分析
  │   - 正式程度评估
  │   - 客观性表达方式
  │   - 模糊语言使用
  └─ 专业表达分析
      - 术语解释方式
      - 新术语引入手法
      - 专业性与可读性平衡
    ↓
[步骤4] 生成风格指南
  - 风格特征总结
  - 写作模板库
  - 术语库
  - 常用表达库
  - 应用示例
    ↓
[步骤5] 保存风格指南
  - 保存到 workspace/style-guides/
  - 格式：Markdown文档
  - 文件名：{领域}-style-guide.md
```

## 六、常用表达库

### 6.1 过渡表达
- 递进：此外、而且、更重要的是
- 转折：然而、但是、不过
- 因果：因此、所以、由此可见
- 总结：综上所述、总而言之、总之

### 6.2 强调表达
- 重要性：值得注意的是、需要指出的是、关键在于
- 确定性：显然、无疑、确实
- 程度：非常、极其、显著

### 6.3 限定表达
- 可能性：可能、或许、大概
- 范围：一般情况下、通常、在大多数情况下
- 程度：在一定程度上、相对而言、一定程度上

### 6.4 引用表达
- 直接引用：{作者}指出...、{作者}认为...
- 间接引用：根据{作者}的研究...
- 对比引用：与{作者}不同的是...

## 文章收集规则

### 用户指定文件

1. 用户上传文件或粘贴文本
2. 直接使用用户提供的内容
3. 解析文章结构和内容

### AI自动搜索

1. 用户指定领域/关键词
2. AI搜索相关文章（3-10篇）
3. 筛选高质量文章
4. 用于风格分析

### 混合模式

1. AI先搜索基础文章
2. 用户补充指定文章
3. 综合分析所有文章

## 风格指南应用

### 应用时机

撰写模块调用时，检查是否有对应的风格指南：
- 如果有 → 展示风格指南摘要，询问用户是否应用
- 如果没有 → 询问用户是否需要学习风格

### 应用方式

- 用户说"应用" → 应用风格指南到撰写
- 用户说"不应用" → 使用默认风格撰写
- 风格指南作为参考，保持风格一致性

## 输出格式

风格指南保存为Markdown格式，文件名格式：`{领域}-style-guide.md`

保存位置：`workspace/style-guides/`
