# 校对模块

## 功能说明

灵活的校对功能，支持用户自定义校对方向和要求。

## 触发词

"检查"、"校对"、"审核"、"验证"、"修改建议"

## 校对流程

```
用户触发校对功能
    ↓
[步骤1] 询问校对方向
  - 请选择校对维度（可多选）：
    □ 用词规范（术语准确性、用词一致性）
    □ 文章结构（段落组织、逻辑层次）
    □ 用句要求（句式规范、长短句搭配）
    □ 语法检查（错别字、语病、标点）
    □ 格式规范（标题层级、编号、字体）
    □ 法律条文引用（法条编号、引用格式）
    □ 专利特殊要求（权利要求格式、充分公开）
    □ 全部检查
    ↓
[步骤2] 执行校对
  - 按用户选择的维度进行校对
  - 生成针对性的校对报告
    ↓
[步骤3] 输出结果
  - 校对报告（按维度分类）
  - 修改建议（具体到行）
  - 可选：直接生成修改后的文档
```

## 校对维度

### 1. 用词规范

**检查内容**：
- 术语准确性：是否使用正确的专业术语
- 用词一致性：同一概念是否使用相同术语
- 专业术语使用：术语是否符合行业规范

**常见问题**：
- 术语不一致（如"处理器"和"处理单元"混用）
- 术语错误（如将"电阻"误写为"电阻器"）
- 用词不当（如"进行"、"实现"等滥用）

**修改建议格式**：
```
第X行："{原文}" → 建议修改为 "{修改后}"
原因：[术语不一致/术语错误/用词不当]
```

---

### 2. 文章结构

**检查内容**：
- 段落组织：段落长度是否合理，主题是否集中
- 逻辑层次：论述是否清晰，层次是否分明
- 章节安排：章节顺序是否合理，过渡是否自然

**常见问题**：
- 段落过长（超过200字）
- 逻辑跳跃（缺少过渡）
- 层次混乱（论述顺序不合理）

**修改建议格式**：
```
第X段：[问题描述]
建议：[修改建议]
```

---

### 3. 用句要求

**检查内容**：
- 句式规范：句子结构是否完整、正确
- 长短句搭配：是否合理搭配长短句
- 从句使用：从句使用是否恰当

**常见问题**：
- 句子过长（超过50字）
- 句子残缺（缺少主语或谓语）
- 从句嵌套过深（超过3层）

**修改建议格式**：
```
第X行：[问题描述]
原句：[原句]
建议：[修改后的句子]
```

---

### 4. 语法检查

**检查内容**：
- 错别字：是否有错别字
- 语病：是否有语病（搭配不当、成分残缺等）
- 标点符号：标点使用是否正确

**常见问题**：
- 错别字（如"的"、"地"、"得"误用）
- 语病（如"通过...使..."句式）
- 标点错误（如逗号、句号误用）

**修改建议格式**：
```
第X行：[问题描述]
原句：[原句]
建议：[修改后的句子]
```

---

### 5. 格式规范

**检查内容**：
- 标题层级：标题层级是否正确
- 编号规范：编号是否连续、格式是否统一
- 字体要求：字体、字号是否符合要求

**常见问题**：
- 标题层级跳跃（如H1直接到H3）
- 编号不连续（如1、2、4，缺少3）
- 格式不统一（如编号格式不一致）

**修改建议格式**：
```
第X处：[问题描述]
建议：[修改建议]
```

---

### 6. 法律条文引用

**检查内容**：
- 法条编号准确性：引用的法条编号是否正确
- 引用格式：引用格式是否规范
- 时效性：引用的法条是否为最新版本

**常见问题**：
- 法条编号错误（如"专利法第22条"误写为"专利法第23条"）
- 引用格式不规范（如缺少条款号）
- 引用过时法条（如引用已修改的法条）

**修改建议格式**：
```
第X处：[问题描述]
原引用：[原引用]
建议：[修改后的引用]
```

---

### 7. 专利特殊要求

**检查内容**：
- 权利要求格式：是否符合专利法要求
- 说明书充分公开：是否充分公开技术方案
- 新颖性/创造性表述：是否恰当表述

**常见问题**：
- 权利要求格式错误（如前序部分和特征部分混淆）
- 说明书不充分（如缺少关键技术细节）
- 表述不当（如"首创"、"唯一"等绝对化表述）

**修改建议格式**：
```
第X处：[问题描述]
建议：[修改建议]
```

---

### 8. 全部检查

执行以上所有维度的检查，生成综合校对报告。

## 校对规则

| 规则类型 | 说明 | 优先级 |
|----------|------|--------|
| **必须修改** | 法律条文引用错误、权利要求格式错误、严重语法错误 | 高 |
| **建议修改** | 用词不当、格式不规范、逻辑不清晰 | 中 |
| **可选修改** | 风格优化、表达优化、冗余内容简化 | 低 |

## 输出格式

校对报告保存为Markdown格式，文件名格式：`proofread-{文档名称}-{日期}.md`

保存位置：`workspace/proofread-reports/`
